# Content Guide

How to read, navigate, and get the most from SignalStack.

---

## Who This Repo Is For

| Audience | Start Here |
|----------|-----------|
| **Development researchers** working in climate, health, gender, or food security | [Newsletter Issues](../issues/) for curated report summaries |
| **M&E practitioners** looking for tools and frameworks | [Featured Tools](../featured/) and [InsightStack](https://github.com/Varnasr/InsightStack) |
| **Data science learners** wanting visual, concept-based study aids | [101 Data Drawings Extras](../extras/books/101_data_drawings_extras/) |
| **Students and early-career researchers** exploring development topics | Start with any newsletter issue and follow the source links |

---

## How Content Is Organised

### Newsletter Issues (`issues/`)

Each issue folder contains a `README.md` with:

1. **Publication date and topic categories**
2. **Featured items** — each with a 100-200 word summary explaining what the resource is, what it found, and why it matters
3. **Key takeaways** — one actionable insight per featured item
4. **Link to the full Substack issue** — for the complete newsletter experience
5. **Cross-references** — to extended notes and related content in this repo

Issues are numbered chronologically. The most recent issue appears first in the [main README](../README.md).

### Featured Tools (`featured/`)

Each featured tool gets its own markdown file with:

- Tool name, author, and link
- What it covers (with a topic breakdown table)
- Who should use it and why it matters
- Limitations and caveats

### Book Companions (`extras/books/`)

Structured study aids organised by book:

| File | Purpose |
|------|---------|
| `summary.md` | Book overview, audience, and structure breakdown |
| `key_concepts.md` | Definitions table with "When It Matters" context |
| `applications.md` | Real-world use cases with examples and common pitfalls |
| `visual_walkthroughs.md` | Video links with descriptions and a suggested learning workflow |
| `resources.md` | Further reading: books, courses, tools, and related repos |

### Extended Notes (`extras/issue notes/`)

Deep-dive reference files that accompany newsletter issues. Each note contains source links, follow-up reading, and connections to other OpenStacks repositories.

## How to Cite Content

If you reference specific content from this repo:

```text
Sri Raman, V. (2024). SignalStack: Research Rundown Newsletter Archive.
https://github.com/Varnasr/SignalStack
```

A machine-readable citation is available in [CITATION.cff](../CITATION.cff).
